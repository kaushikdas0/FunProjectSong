1. Create `src/components/Icon/Icon.tsx` -- a thin wrapper around pixelarticons:

```tsx
import {
  Plus,
  Download,
  Reload,       // or Refresh -- check pixelarticons exports
  Zap,          // brand mark candidate
  Heart,        // decorative candidate
} from 'pixelarticons/react';

// NOTE: Check actual export names from pixelarticons. The research says
// Refresh/Download/Plus exist. Verify import names work. If 'pixelarticons/react'
// does not export these names, check the package README or try individual imports
// like 'pixelarticons/react/Download'.

type IconName = 'generate' | 'download' | 'regenerate' | 'brand' | 'decorative';

const iconMap = {
  generate: Plus,
  download: Download,
  regenerate: Reload,
  brand: Zap,
  decorative: Heart,
} as const;

interface IconProps {
  name: IconName;
  size?: number;
  className?: string;
}

export function Icon({ name, size = 24, className = '' }: IconProps) {
  const Component = iconMap[name];
  return <Component width={size} height={size} className={className} />;
}
```

IMPORTANT: pixelarticons uses `fill="currentColor"` by default. Color is controlled entirely via CSS `color` property (Tailwind `text-*` classes). NEVER set fill color directly on the SVG.

Render icons at multiples of 24 (24, 48) for pixel-perfect sharpness per RESEARCH.md pitfall 5.

3. Update `src/App.tsx` to add routing with dev-only kitchen sink:

```tsx
import React from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import MainScreen from './screens/MainScreen';

const KitchenSinkScreen = import.meta.env.DEV
  ? React.lazy(() => import('./screens/KitchenSinkScreen'))
  : null;

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<MainScreen />} />
        {import.meta.env.DEV && KitchenSinkScreen && (
          <Route
            path="/kitchen-sink"
            element={
              <React.Suspense fallback={null}>
                <KitchenSinkScreen />
              </React.Suspense>
            }
          />
        )}
      </Routes>
    </BrowserRouter>
  );
}
```

This ensures /kitchen-sink is tree-shaken from production builds.
