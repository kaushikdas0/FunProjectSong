  <action>
Audit every element in MainScreen.tsx and ComplimentCard.tsx for mobile overflow at 375px width (with 16px padding each side = 343px content area).

Known risk areas from research:
1. **ComplimentCard max-width**: Currently `max-w-[400px]`. At 375px viewport with `px-4` parent padding, the card wrapper provides 343px. The card has `w-full` so it shrinks, but verify the card's own `p-[10px]` outer padding does not cause issues. If the 400px max-width causes any layout weirdness, change to `max-w-sm` (384px) which still shrinks gracefully.

2. **Card wrapper in MainScreen**: The card section uses `max-w-sm` (384px). On 375px viewport this is fine — `w-full` kicks in.

3. **Title font size**: `text-5xl` (3rem = 48px) may be too large on very small screens. If it causes wrapping that looks bad, add `sm:text-5xl text-4xl` to scale down on mobile.

4. **Outer container**: Verify `min-h-screen` allows scrolling when content (input + button + card + download button) exceeds viewport height. The outer div has `overflow-hidden` — this MUST be changed to `overflow-x-hidden overflow-y-auto` if content can exceed viewport height on short screens. Otherwise the card and download button get clipped.

5. **Download button**: Verify it does not overflow the card wrapper width.

Apply fixes as needed. If no changes are needed for a file, do not modify it. The goal is a working mobile layout, not changes for the sake of changes.

IMPORTANT: Change the outer MainScreen div from `overflow-hidden` to `overflow-x-hidden` so vertical content can scroll on short viewports while still preventing horizontal scroll from absolute-positioned icons. Also change `min-h-screen` layout: keep `min-h-screen` but replace `justify-center` with `justify-start pt-12 sm:justify-center sm:pt-0` so on mobile the content starts near the top (allowing scroll) while on desktop it remains vertically centered.
